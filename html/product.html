IIDAXE TOP(概要)

とあるゲームの一分野における難易度比較をヒエラルキー形式で考察, 提言, 表現する為のツール

機能説明

    基本操作

  • VERSIONセレクトボックス変更でMUSIC & SCOREセレクトボックス内容が変更
  • Generateボタンにより MUSIC & SCORE選択中の内容でドラッグ可能なボックスを生成
  • Save, Loadボタンにより状態の保存, 読出(Local Storage使用)
  • 保存, 読出を行う内容は、"生成および配置した位置", "TARGET LEVEL, OPTIONセレクトボックスの選択情報"
  • Initializeボタンで全て削除(保存内容も削除)
  • Shiftキー+Ctrlキーの同時押ししながらボックスをダブルクリックで対象のみ削除(セレクトボックス内に再挿入)
  • Altキー押しながらダブルクリックで対象のボックスのTextageの譜面ページへ別ウィンドウにて移動
  • マウスオーバーで吹き出しによる情報表示

    • 軸, 座標について

  • Y軸 : 対象譜面において設定した目標の達成難易度(上部 達成難度高[difficult], 下部 達成難度低[easy])
  • X軸 : 対象譜面において特異であるか(左部 特異である[singular], 右部 一般的である[ordinary])
    • (同じY座標にて、同程度の難易度ではあると思っても個人差, 譜面の特異性, 特徴から一般的で無い と考えるならば左部へ)

    Config Searchボタン 及びモーダル

  • 配置済みのものの位置を検索, 選択しSerchボタンにてその位置に表示を移動
  • Name Saveにて作成者名保存しページタイトルに反映
  • 作成した内容をTwitter共有用URLを作成しTweetボタンにて投稿可能

    • その他

  • Informationボタンで当説明と同内容の表示

追加実装予定

  • 検索移動後、対象の表示を一定時間目立たせる
  • 配置済み, 未配置一覧をモーダルで表示

将来対応予定

  • 配置可能領域設定(現在 背景画像で固定してしまっている)
  • 軸の表示変更設定(同上)
  • マルチセレクト可能なセレクトボックスで一括生成
  • MUSIC & SCORE要素を追加対応した際の処理(主に配置済みかの判定の仕様について)
  • スマートフォン等からのブラウザ対応

作成環境等

  • html, JavaScript, jQuery
  • Windows7, Node.js, Visual Studio Code
